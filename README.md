# Выпускная работа по курсу "Java-разработка" Вконтакте
# Speechmood —  это пользовательский аналог современной социальной сети, разработанный для обмена контентом, текстовыми сообщениями и создания тематических сообществ.

> API поддерживает полный спектр CRUD-операций, позволяя не только создавать контент, но и точечно обновлять данные профиля, включая загрузку аватаров. Для социального взаимодействия реализованы механизмы подписок и асинхронная система лайков, работающая без перезагрузки страниц. Универсальная сущность сообщений позволяет использовать один и тот же программный код как для чатов в группах, так и для комментариев под постами.
>
> Бизнес-логика опирается на пять ключевых сущностей: пользователи, профили, посты, сообщения и группы. Сервер обеспечивает строгую приватность контента, гарантируя, что публикации пользователей видны исключительно их подтвержденным подписчикам. Система групп позволяет создавать изолированные пространства для общения, где владельцы групп могут полностью управлять составом участников.
> 
> Центральным звеном безопасности является связка Spring Security и JWT, работающая по схеме с Access и Refresh токенами для обеспечения авторизации без сохранения состояния сессии. Взаимодействие с данными реализовано через Spring Data JPA и Hibernate, что позволяет гибко управлять схемой базы данных и сложными связями между сущностями в PostgreSQL.

---

## Технологический стек

### Backend
* **Язык и фреймворк**: Java 17, Spring Boot 3.
* **Безопасность**: Spring Security + JWT.
* **Data Layer**: Spring Data JPA & Hibernate.
* **База данных**: PostgreSQL.

### Frontend
* **Core**: HTML5, CSS3, Vanilla JavaScript.
* **Network**: Асинхронное взаимодействие через Fetch API.
* **Storage**: Хранение токенов в localStorage.

---

## Структура данных

* **User & Profile**: Личные данные, аватарки и система подписок.
* **Post**: Публикации в ленте, доступные только друзьям.
* **Group**: Пользовательские чаты и сообщества.
* **Message**: Универсальная сущность для чатов и комментариев под постами.
* **Like**: Система социального одобрения контента.

---

### Аутентификация и Пользователи
* `POST /api/auth/register` — Регистрация нового аккаунта.
* `GET /api/users` — Список всех пользователей системы.
* `PUT /api/users/{id}/update` — Редактирование профиля.
* `GET /api/users/{id}` — Получение детальной информации о конкретном пользователе по его ID.
* `PUT /api/users/{id}/update` — Обновление данных профиля (имя, email, возраст, пол).
* `DELETE /api/users/{id}/delete` — Полное удаление пользователя из системы.
* `PUT /api/users/change-password` — Смена пароля авторизованным пользователем.

### Посты и Лайки
* `POST /api/posts/create` — Создание новой текстовой публикации.
* `GET /api/posts` — Загрузка ленты постов.
* `POST /api/posts/{id}/like` — Поставить или убрать лайк.
* `DELETE /api/posts/{id}/delete` — Удаление собственного поста.

### Группы и Сообщения
* `POST /api/groups/create` — Создание новой группы.
* `POST /api/groups/{id}/members/add` — Добавление нового участника в существующую группу.
* `POST /api/groups/{id}/members/remove` — Исключение участника из группы.
* `POST /api/messages/create` — Универсальный эндпоинт для отправки текста. Используется и для сообщений в группах, и для комментариев под постами.


### Друзья
* `POST /api/profiles/friends/add` — Подписка на пользователя.
* `POST /api/profiles/friends/remove` — Отписка от пользователя.
* `GET /api/profiles/friends/list` — Получение списка всех профилей, на которые подписан текущий юзер.
